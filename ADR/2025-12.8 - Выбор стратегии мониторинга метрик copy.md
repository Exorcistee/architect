
**ADR: 2025-12.8 - Выбор стратегии мониторинга метрик**

**Дата:** 2 декабря 2025 г.

**Статус:** Принято

**Контекст:**

Чтобы наша платформа для онлайн-консультаций хорошо работала с деньгами и личными данными пользователей, нужно решить, как подойти к безопасности, чтобы защитить информацию и соответствовать законам.

**Рассмотренные варианты:**

**Snyk + OWASP ZAP**: Комбинация для проверки уязвимостей в зависимостях и веб-безопасности.
**SonarQube + Burp Suite**: Классическое решение для статического анализа кода и пентестинга.
**GitLab SAST + Dependency Scanning**: Встроенные инструменты безопасности GitLab.
**Ручные проверки**: Проверять безопасность вручную без специальных инструментов.

**Решение:**

Выбрана комбинация Snyk + OWASP ZAP для автоматической безопасности.

**Обоснование:**

- **Простая интеграция:** Snyk легко подключается к GitHub, OWASP ZAP работает из коробки.
- **Хорошо для стартапа:** Не требует сложной настройки, есть бесплатные тарифы для начала.
- **Конкретные отчёты:** Показывает именно те уязвимости, которые есть в нашем коде.
- **Автоматизация:** Можно встроить в CI/CD и проверять каждый коммит автоматически.
  
**Недостатки**

- **Не покрывает всё**: Нужны дополнительные инструменты для полного аудита безопасности.
- **Ложные срабатывания**: Иногда показывает проблемы там, где их на самом деле нет.
- **Требует времени на настройку**: Чтобы инструменты не мешали разработке, нужно правильно их настроить.

**Последствия:**

- Настроим Snyk на автоматическую проверку зависимостей npm.
- Интегрируем OWASP ZAP в пайплайн для сканирования API.
- Будем проводить ручные пентесты раз в квартал для критичных функций.
- Создадим процесс быстрого исправления найденных уязвимостей.
